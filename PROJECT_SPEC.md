# Telecom Manager Project Specification

## Назначение проекта

Telecom Manager — внутреннее веб-приложение для учета телеком-подключений, клиентов, материалов, складов, финансов, расходов и дополнительных работ. Система предназначена для монтажников, офиса и администратора. Основная бизнес-логика строится вокруг подключений клиентов и движения материалов между складами.

## Стек

- Python 3.12
- FastAPI
- SQLAlchemy 2.0
- Alembic
- PostgreSQL
- Jinja2
- Bootstrap 5
- HTMX
- Alpine.js
- Docker / docker compose

## Структура базы данных

### users

Пользователи системы. Используются для авторизации, фиксации монтажника в подключениях, автора складских операций и владельца расходов.

Ключевые поля:

- username
- full_name
- hashed_password
- role
- phone
- email
- is_active
- last_login_at

### warehouses

Справочник складов. На текущем этапе используются два склада:

- Эллко
- Оптимасеть

Остатки не хранятся в таблице складов.

### materials

Справочник материалов:

- ONU, единица: штуки
- Кабель витая пара, единица: метры
- Кабель оптика круглая, единица: метры
- Кабель оптика лапша, единица: метры

Остатки не хранятся в таблице материалов.

### inventory_transactions

Единый журнал всех движений материалов. Это единственный источник истины по складу.

Типы операций:

- RECEIPT
- CONNECTION
- TRANSFER_IN
- TRANSFER_OUT
- RETURN
- ISSUE_TO_THIRD_PARTY
- WRITE_OFF
- ADJUSTMENT

Количество может быть положительным или отрицательным. Ноль запрещен. Текущий остаток вычисляется запросом:

```sql
SELECT warehouse_id, material_id, SUM(quantity)
FROM inventory_transactions
GROUP BY warehouse_id, material_id;
```

### clients

Клиенты провайдеров ELLKO и OPTIMASET.

Правила номера договора:

- для ELLKO хранится номер договора;
- для OPTIMASET хранится номер телефона без 8.

Ключевые поля:

- provider
- contract_number
- login
- address
- phone
- comment

### connections

Подключения клиентов. Связаны с клиентом, складом, монтажником, материалами, складскими операциями и финансовыми транзакциями.

Типы подключений:

- NEW
- RECONNECT
- ONU_REPLACE
- CABLE_REPLACE
- WITHOUT_MATERIALS
- CUSTOM

Ключевые поля:

- client_id
- warehouse_id
- connection_type
- connection_date
- installer_id
- price
- office_amount
- installer_amount
- comment

### connection_materials

Материалы, указанные в подключении. Одно подключение может содержать несколько материалов. Каждая строка описывает материал и количество, связанное с подключением.

Фактическое списание склада фиксируется отдельными строками в inventory_transactions с типом CONNECTION.

### finance_transactions

Универсальная таблица финансовых операций. Может относиться к подключению, расходу или дополнительной работе. Для одного финансового события должен быть указан ровно один источник.

Ключевые поля:

- connection_id nullable
- expense_id nullable
- extra_work_id nullable
- amount
- transaction_type
- comment
- created_at

### expenses

Личные расходы пользователя, обычно монтажника. Связанные финансовые транзакции создаются через finance_transactions.

### extra_works

Дополнительные работы как отдельная сущность. Могут быть связаны с клиентом и опционально с подключением.

### event_logs

Журнал событий системы. Используется для аудита действий пользователя и системных операций.

## Правила движения материалов

1. Остатки материалов нигде не хранятся отдельным полем.
2. Остатки всегда считаются суммой inventory_transactions.quantity.
3. Приход создает положительную запись RECEIPT.
4. Перемещение создает две записи:
   - TRANSFER_OUT с отрицательным количеством на складе-источнике;
   - TRANSFER_IN с положительным количеством на складе-получателе.
5. Выдача третьему лицу создает отрицательную запись ISSUE_TO_THIRD_PARTY.
6. Возврат создает положительную запись RETURN.
7. Списание создает отрицательную запись WRITE_OFF.
8. Корректировка создает ADJUSTMENT с положительным или отрицательным количеством.
9. Подключение создает отрицательные записи CONNECTION по каждому списанному материалу.
10. Запрещено создавать операцию, если после нее остаток материала на складе станет отрицательным.

## Правила финансов

Финансовые операции хранятся в finance_transactions. Подключение автоматически создает финансовую транзакцию с типом CONNECTION_PAYMENT.

## Распределение денег

Для подключения действуют правила:

- цена 1000: монтажник 1000, офис 0;
- цена 3000: монтажник 1000, офис 2000;
- произвольная сумма: монтажник и офис вводятся вручную, сумма долей должна равняться цене подключения.

## Права пользователей

Роли:

- ADMIN: полный доступ, включая удаление подключений;
- OFFICE: рабочий доступ для офисных операций;
- INSTALLER: рабочий доступ монтажника.

На текущем этапе реализована авторизация и ограничение удаления подключений только для ADMIN.

## Структура отчетов

Раздел отчетов зарезервирован в навигации. Планируемая структура:

- отчет по подключениям за период;
- отчет по материалам и остаткам;
- отчет по движениям материалов;
- финансовый отчет по подключениям;
- отчет по расходам;
- отчет по допработам;
- отчет по монтажникам.
